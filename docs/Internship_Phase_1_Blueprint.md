# Phase 01 — Blueprint

**Organization:** Rooman Technologies — Internship Project Phase Guide  

**Timeline:** Week 1 of the project phase · **5 working days**

**Phase principle:** Plan before you build.

No code and no tangible project output proceed until the problem is fully understood **and** the blueprint is approved by your trainer.

---

## Goal of this phase

You will produce an **approved Blueprint Document** that proves you know:

- What problem you are solving and for whom  
- How you will approach it (conceptually — not implementation detail)  
- What tools and data you will use  
- How you will know you succeeded  

**Gate to Phase 2 (Build):** Your trainer must **sign off on the Blueprint Document** before anyone starts building.

---

## Universal checklist — what every intern does

Complete these steps during Blueprint week, regardless of track.

| Step | What to do |
|------|------------|
| **1. Form your team** | Work in a team of **2–3 interns**, or solo if assigned. Confirm structure with your trainer by **morning of Day 1**. |
| **2. Define the problem** | Write a **single clear sentence** problem statement: who is affected and what goes wrong today. |
| **3. Sketch the solution** | Describe your **approach** — the thinking behind the solution — **not** the code and **not** the final artifact. |
| **4. Confirm your stack** | List **every** tool, dataset, API, and technology you plan to use. Verify you **have access** (accounts, licenses, data). |
| **5. Set success criteria** | Define **three measurable outcomes** using the pattern: “We will know this worked if …”. Avoid vague goals. |
| **6. Build a day plan** | Map **Phase 2 (Build)** and **Phase 3 (Present)** **day-by-day**: what finishes each day so you hit deadlines. |

---

## Domain-specific deliverables (before Phase 2 begins)

Bring these items into your Blueprint so your trainer can approve a concrete plan for your track.

| Track | You must produce (in blueprint) |
|-------|--------------------------------|
| **IT Ignite** | Problem brief · tech stack choice · GitHub repo initialised · system sketch or wireframe |
| **AI / ML** | Use-case defined · dataset sourced · modelling approach chosen · evaluation metric chosen |
| **AI Data Analyst** | Dataset sourced · problem statement · hypothesis · analysis plan incl. expected charts |
| **Web Dev** | App brief · user stories · architecture diagram · DB schema draft · Figma/Excalidraw wireframe |
| **IT-Ops** | Infrastructure problem defined · tooling choice (e.g. Ansible / Terraform / Prometheus) · architecture diagram |
| **Cybersecurity (CS-EH)** | Scope · threat model or risk-assessment template · tools list · assessment methodology |

---

## Definition of done — Phase 1

Phase 1 is complete when:

1. The **Blueprint Document** includes the universal checklist (problem, sketch, stack, metrics, phased day plan).  
2. Track-specific deliverables above are filled in enough that Phase 2 has **zero ambiguity**.  
3. Your trainer records **explicit approval** to move to Phase 2 — **Build**.
